# Instructions for agents working on this repo

- **Never test or verify visual/UI changes yourself** — no starting a local
  dev server, no opening the site in a browser, no screenshotting — unless
  the user explicitly asks for it. The user (or whoever owns this repo)
  wants to be the one checking visual results. Just report what changed
  and stop.

- **Commit and push on behalf of the user.** This repo is run by a designer
  with no git experience. When she says to push, do it for her: `git add`,
  `git commit`, and `git push`. Write commit messages describing what was
  done — do not mention the agent or add co-author trailers — and keep them
  concise and concrete (a normal length, neither too long nor too short).
